> **依据文档：** `ocr-engine-unification.v0.5-AMEND.md`（D4 节）

## D4 —— 三级编号异常检测（域评估）

**状态：`- [ ] 有条件通过`**

**评估：** D4 的设计方向正确——对"超出预期层级"的编号做标记是合理的质量门。但当前方案隐含一个硬编码假设（`expected_depth=2`），该假设在《秘方求真》上偶然成立，对 KZOCR 将要处理的其他 TCM 书不成立。**若不修正，D4 在 KZOCR 部署场景下会产生大规模误报。**

---

### 1. 《秘方求真》中三级编号的真实性质

#### 1.1 编号方案

《秘方求真》使用 **2 段编号**：`节号.方序号`（如 `16.7` = 第 16 节、第 7 方）。该书不含章号前缀。

#### 1.2 实测数据

| 指标 | 数值 |
|------|------|
| 方剂总数 | 1723 |
| 三级编号实例（如 `16.7.1`） | 4 例（0.23%） |
| 其中 `16.7.x` 系 | 至少包含 `16.7.1`, `16.7.2` |

**TOC 项目对这 4 例的处理方式**：
- 归类为"小节级偏差"，与"书籍编辑错误"并列
- 写入 `hierarchy_anomaly` 表，`resolution = pending`
- **它们被成功入库**（并非被拒绝）——OCR-BUG-010 修复后，三级编号可直接被正则 `\d{1,3}\.\d{1,3}(?:\.\d{1,3})?` 捕获并正常入库
- 其中 `16.7.2` 已知"无组成字段"（遗留问题），暗示该条目可能是书中编辑不完整的内容

#### 1.3 它们是真正的异常吗？

**结论：对《秘方求真》而言，三级编号是"值得标记但不需阻断"的异常。** 理由：

- **可能是合法的子方/加减方**：TCM 书籍中，一首主方后列加减变化（如"寒甚加附子，气虚加黄芪"）是标准写法，但少数书用子编号标记这些变化。`16.7.1`、`16.7.2` 极可能是第 16.7 方的两个子方。
- **也可能是排版错误**：原书排版中，某些方剂编号的起始数字被误叠印，OCR 识别为三级编号。
- **0.23% 的低频次**使人工复核成本极低（4/1723 ≈ 每人 2 分钟可查完）。

**在 TOC 项目语境下，D4 行为合理**：标记→人工确认→分类为"合法子方"或"排版错误"。

### 2. 对 KZOCR 的适用性分析

#### 2.1 核心问题：`expected_depth` 不可硬编码为 2

D4 方案中的 `check_hierarchy_anomaly` 逻辑：

```python
def check_hierarchy_anomaly(text: str, page_num: int) -> list[HierarchyAnomaly]:
    """扫描 OCR 文本中段数 > 2 的编号，返回异常列表。"""
```

隐含假设：**所有 TCM 书的正常方剂编号都是 2 段**。

这在 KZOCR 的部署场景中**不成立**。KZOCR 是通用 TCM OCR 系统，会处理多种编号方案的书籍：

| 编号方案 | 示例 | 正常段数 | 应用场景 |
|---------|------|---------|---------|
| `节号.方序号` | `16.7` | 2 | 《秘方求真》式单卷经验方集 |
| `章号.方序号` | `1.7` | 2 | 单章教材，同一章内排列 |
| `章号.节号.方序号` | `1.2.3` | 3 | **多卷本中医书**（极为常见） |
| `章号.节号.子节号.方序号` | `1.2.3.4` | 4 | 大型方书（如《中华本草》《普济方》） |

以 **3 段为正常格式**的 TCM 书籍非常普遍。例如：
- 使用"卷一、卷二"分章的书籍常以 `1.1.1`（卷一.第一节.第一方）作为编号
- 院校教材《方剂学》中，章节结构通常为"章→节→方"三层次
- 临床医案类书籍：`患者编号.诊次.方序号`

#### 2.2 假设误判的后果

如果将上述 3 段书籍送入 D4，**每一首方剂都会被标记为异常**。对于一本 1000 方剂的书，D4 会产生约 **600-1000 个假阳性记录**（取决于编号格式分布）。这远远超出人工复核的可行范围。

#### 2.3 低频假阳性（与 C1 比较）

D4 和 C1 都涉及"人工复核"，但规模不同：

| 检测项 | 《秘方求真》数据 | 人工工作负荷 |
|-------|----------------|------------|
| C1 跨页泄露 | 20/970 页（2%） | 20 个实例，每个需检查截断是否正确 |
| D4 三级编号 | 4/1723 方（0.23%） | 4 个实例，每个需对照原书确认 |
| D4 假阳性（若 expected_depth=2 误用于 3 段编号书） | **600-1000+/1000 方** | **彻底不可行** |

C1 的误报率是可管理的（预期 ≤2-5%），因为探针设计恰好规避了 TCM 书的重复文本陷阱。D4 的误报率在编号方案不匹配时是**灾难性的**（接近 100%）。

### 3. TCM 域特定洞察

#### 3.1 何时三级编号是合法内容

以下场景为 TCM 书中三级编号的**合法用途**，不应标记为异常：

1. **加减方/变方**：主方后列出的变化条目。如 `8.5` 主方下有 `8.5.1`（气虚重加黄芪）、`8.5.2`（血虚重加当归）——这是 TCM 临床处方的标准记录方式。
2. **章-节-方自然结构**：书籍正文的自然编号，`章号.节号.方序号` 是最常见的三级结构。
3. **多级分类索引**：如《中医大辞典·方剂分册》等工具书使用 `类别.亚类.方号` 的多级编号。

#### 3.2 何时三级编号是真正的错误

以下场景中三级编号更可能是 OCR 失真或排版错误：

1. **方剂编号中混入 OCR 噪声**：如 `16.`（末尾小数点被误读）导致正常编号 `16.7` 变为 `16..7` 或 `16.7.`
2. **页码残留**：页眉/页脚数字被卷入正文，出现在方剂编号附近，被误解析为编号的一部分
3. **双层编号的退化形式**：某些书的编号原为 `16-7`（连字符分隔），OCR 模型中 `.` 与 `-` 形近致 `16.7` 被追加另一段
4. **排版叠印**：古书籍排版中编号与修饰符（如圈码、黑底白字标记）重叠，产生额外识别段

#### 3.3 区分策略（TCM 领域知识建议）

要区分上述合法与非法三级编号，不能仅靠段数计数。推荐补充以下启发式（而非替代 D4）：

| 启发式 | 判断逻辑 | 可靠性 |
|--------|---------|--------|
| **编号连续性** | 如果编号 `N.1` ~ `N.k` 连续且 N 是已知合法编号，则 `N.i` 更可能是合法子方而非错误 | 中高 |
| **内容结构完整度** | 三级编号条目若包含完整字段（组成/用法/功用等），更有可能为合法子方；缺少关键字段者则是 OCR 错误 | 中 |
| **跨页位置** | 三级编号出现在页末/页首过渡区，更可能是 OCR 跨页泄露导致的编号断裂 | 中 |
| **TOC 匹配** | 如果书籍的目录（TOC）中列出的编号含三级结构，则整个书的编号方案为三级，D4 不应触发 | 高 |

### 4. 建议

#### 4.1 【必要】`expected_depth` 参数化

将 `expected_depth` 从硬编码值改为 **书籍级配置**，从 TOC 分析结果或配置中读取：

```python
@dataclass
class HierarchyAnomalyConfig:
    expected_depth: int = 2           # 该书的正常编号段数（从 TOC 分析推断）
    check_enabled: bool = True        # 可独立关闭

def check_hierarchy_anomaly(
    text: str,
    page_num: int,
    config: HierarchyAnomalyConfig,
) -> list[HierarchyAnomaly]:
    """扫描 OCR 文本中段数 > config.expected_depth 的编号，返回异常列表。"""
```

**默认值 `expected_depth=2` 保留**（当前 KZOCR 默认处理单卷经验方集），但不应作为唯一样式。

#### 4.2 【必要】TOC 驱动的 `expected_depth` 推断

如果 KZOCR 未来集成 TOC 分析模块（如 v0.5 提到的 `toc_analyzer.py`），应让 TOC 分析结果自动反推 `expected_depth`：

```
从 TOC 中所有编号的段数统计:
  - 若 90%+ 编号为 2 段 → expected_depth = 2
  - 若 90%+ 编号为 3 段 → expected_depth = 3
  - 混合模式 → 取众数，并记录偏离值
```

在 TOC 模块尚未实现时，可通过 CLI 参数 `--expected-depth 3` 配置。

#### 4.3 【建议】输出格式增强

当前 D4 输出仅一条记录。建议增加**上下文字段**：

```python
@dataclass
class HierarchyAnomaly:
    recipe_no: str
    depth: int
    source_page: int
    resolution: str = "pending"
    # 新增
    surrounding_text: str = ""      # 该编号前后 50 字上下文（便于快速判断）
    ocr_confidence: float | None = None  # 该字符的 OCR 置信度（若有）
```

`surrounding_text` 大幅提升人工复核效率——审校员无需打开原书，仅看上下文即可判断是合法子方还是 OCR 噪声。

#### 4.4 【建议】P3 优先级的再确认

D4 标记为 P3（低优先）是合理的，但理由应补充领域上下文：

- 对于《秘方求真》类单卷经验方集（2 段编号）：D4 覆盖率 ≈0.23%，价值有限但成本也很低
- 对于多卷本 TCM 书（3 段编号为正常格式）：D4 不可用（因 `expected_depth` 未参数化）
- **在 `expected_depth` 参数化 + TOC 驱动推断完成前，D4 不宜作为通用异常检测投入生产**

建议将 D4 的计划拆分为两步：

| 步骤 | 内容 | 优先级 | 前提条件 |
|------|------|--------|---------|
| 4a | `expected_depth` 参数化 + 基础实现 | P3 | 无 |
| 4b | TOC 驱动自动推断 + 上下文增强 | P4 | TOC 分析器可用后 |

### 5. 总结评分

| 评审项 | 状态 | 影响 | 优先级 |
|--------|------|------|--------|
| D4：三级编号是否真正异常（《秘方求真》语境） | `- [x] 通过` | 在该书语境下合理：4/1723 例需标记，0.23% 可管理 | — |
| D4：`expected_depth` 硬编码 2 | `- [ ] 需修订` | **关键阻塞问题**：在多段编号 TCM 书中将产生灾难性假阳性 | **P0（方案修正）** |
| D4：假阳性对运营的影响 | `- [x] 通过（有条件）` | 在参数化后非常低（<1% 方剂量级）；硬编码 2 则为毁灭性（近 100%） | — |
| D4：P3 优先级定位 | `- [x] 通过` | 合理，拆分 4a/4b 后 P3+P4 更清晰 | P3 |

**总体领域评价：** D4 的设计意图正确——标记超出书籍正常层级结构的编号。当前方案在《秘方求真》的特定编号方案（2 段）上偶然正确，但作为 KZOCR 通用异常检测体系的一部分，`expected_depth` 必须参数化。若不修正此点，D4 在 KZOCR 首次部署到多卷本 TCM 书籍时会产生大规模误报，使 `hierarchy_anomalies.json` 无法使用。

**一句话建议：参数化 `expected_depth` 是 D4 进入 v0.5 的前提条件，此修正在方案层面仅需约 10 行代码变更，但决定了 D4 是从"可用"变成"通用"。**
