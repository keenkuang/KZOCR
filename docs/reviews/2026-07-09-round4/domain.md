# 中医领域视角评审：OCR 引擎统一架构方案（v0.2 · round4）

> 评审人角色：中医文献与术语领域专家（KZOCR）
> 评审对象：`docs/plans/ocr-engine-unification.md`（v0.2，重点 §4.2 RARE 态 / normalize / 形似混淆集、§6 方剂七类 / 毒性告警 / book_type 分流、§8 假设 2/3/6 裁决）
> 输入：round3 `summary.md`（I6/I9、假设 2/3/6 裁决）+ 本人 round3 初稿 `domain.md`（K1–K10）
> 立场：只评审，不改主方案 / 代码。
> 判定口径：**「真闭合」= 机制 + 数据来源 + 边界都已明确可落地**；仅机制写对、数据来源悬空或边界未划清的，判「机制闭合 / 数据悬空」，不记全分。

## 一、结论（round4 总评）

v0.2 在**领域机制层面**对 round3 的 6 项 High/Medium 问题（K1/K2/K3/K5/K7/K8）作了**正确的设计闭合**——RARE 态、normalize、中医专用混淆集、七类字段、毒性红线、book_type 分流、KZOCR 内置白名单，方向全部正确，且逐一对应到 §4/§6/§8 的裁决。

但**真闭合度是「七分设计、三分悬空」**，存在三类未补的缝隙：

1. **资源文件悬空**：v0.2 反复引用的 `variant_map.json` / `confusion_set.json` / `toxic_herbs.json` / 「中医候选字表」**目前库中均不存在**（已 `find` 确认）。机制正确但开局是空表 → RARE 永不触发、混淆集为空、毒性表为空。若定稿即进实施，第一本《本草纲目》影印本会原样复现 round3 K1 的「万条 UNKNOWN 淹没」。**这是 round4 最该补的一条**。
2. **RARE 误放行 + 闭环毒化风险**：RARE 免人工、且闭环回填若自动化，会把 OCR 错字当生僻药名写回白名单，形成「错→放→扩」正向毒化环（新引入问题 N1/N2）。
3. **两条边界未划清**：①「中医候选字表」与内置白名单的边界（K1 延伸）；② TOC 缺失的影印 / 手写本最小小节 fallback（K5 延伸）；③ 剂量「原串保留」与「用量红线可判定」自相矛盾（K7 延伸）。

**总体**：有条件通过（领域视角）。进入阶段 3/5 前，必须先把 4 个领域资源文件（含种子数据）落地、给 RARE 加置信门 + 抽检、并补两条 fallback 规则，否则 round3 领域灾难会在 v0.2 下以「更隐蔽」形态重演。

## 二、round3 问题闭合度（逐条）

### K1 罕见中医字误判 UNKNOWN（§4.2 步骤 3 → RARE 态）— 机制闭合 / 数据悬空
- **v0.2 做法**：§4.2 步骤 3「归一后不在已知集、但属合法 CJK Ext-A/B 且命中『中医候选字表』→ `RARE`（罕见但允许，不进人工队），`auditSource=rare_allowlist`」。方向与我在 round3 建议 A.2 完全一致。
- **闭合度：机制正确，数据来源未定义**。
  - 「中医候选字表」从哪来？与内置白名单（PASS 集）边界如何划？方案 §4.1 只说「内置精简字形白名单为事实源 + 预计算 rare 候选字表」，但**未说明候选字表的首批来源**（是随包内置种子，还是靠闭环回流冷启动？）。若开局为空，RARE 永不触发，等于 K1 未解。
  - 边界风险：白名单（归一后命中→PASS）与 RARE 候选表（CJK Ext-A/B 命中→RARE）应**互斥且 RARE ⊇ 白名单之外的生僻合法字**。需明确「白名单 = 高频已知；候选表 = 低频但合法且属中医域」，避免把任意 Ext-A/B 合法字都放行（见 N1）。
- **判定：真闭合度 70%**——设计对，缺种子数据 + 边界定义。

### K2 缺繁简 / 异体归一化（§4.2 步骤 1 normalize）— 机制闭合 / 覆盖存疑
- **v0.2 做法**：§4.2 步骤 1「`normalize(c)`：繁→简 + 异体→正体（`variant_map.json` 仅收明确等价项）+ 全角 / 旧字形归正」。对应建议 A.1。
- **闭合度：机制正确，但「仅收明确等价项」偏严**。
  - 繁→简由 opencc 类工具覆盖，黃連→黄连、黨參→党参可解，OK。
  - 异体→正体依赖手工 `variant_map.json`：对 朮→术、栝楼→括楼、藊豆→扁豆 等**明确等价项**能解；但「仅收明确等价项」会导致**边缘异体漏判**——如 蘡薁 的异体、菟丝子「菟」与「兔」通用、葶苈「葶」异写等若未录入，归一失败 → 该字 PASS 不了也 RARE 不了，落到 UNKNOWN/FAIL。
  - 建议 `variant_map` 采用「明确等价（自动合并）+ 候选异体（提示不自动合并，送检）」两档，避免过严漏判或过松误并（如 菟丝子 误并为 兔子）。
- **判定：真闭合度 75%**——机制对，覆盖策略需从「仅明确等价」改成「等价 + 候选两档」。

### K3 方剂库漏七类字段（§6 步骤 4）— **真闭合**
- **v0.2 做法**：§6 步骤 4「以规范 `FormulaComposition`（含 alias/root/referenced/crossPage）为准，补齐七类核心字段（`usage/gongyong/zhuzhi/fangjie/fields_json…`），剂量保留原串」。与建议 B 逐字对齐。
- **闭合度：真闭合**。字段名、归库目标（规范 `FormulaComposition`）、原串保留均明确。唯一待补是 `fields_json` 的七类内部键约定（usage/gongyong/zhuzhi/fangjie 已列，余下来源/加减/疗效/附记应进 `fields_json`）——属实现细节，不阻断。
- **判定：真闭合度 100%**。

### K4 khub / term_kb 闭环悬空（§6 步骤 5 + §8 假设 3）— 机制部分闭合 / 回流 gate 缺失
- **v0.2 做法**：§6 步骤 5「闭环回填机制：人工校正 → `CandidateSubmissionBatch` → term_kb / HerbOCRPattern / khub 术语·方剂库管线」；§8 假设 3 裁决「khub 异步可选、不阻塞」。
- **闭合度：机制对，但回流 gate 未定义**，且 term_kb/khub 仍悬空（同 round3 K4）。
  - 正向：主链只写 zai、khub 异步不阻塞，已回避「把不存在依赖写进方案」的坑（与我对假设 3 的立场一致）。
  - 负向：人工校正 → 白名单 的**审核 gate 缺失**。若 `CandidateSubmissionBatch` 自动写回 `rare_allowlist/variant_map/confusion_set`，OCR 错字被当生僻药名固化（见 N2 毒化）。必须有一道「人工复核 / 抽样审核」才落库。
  - term_kb 路径仍依赖 kimi 侧，而我记忆中 kimi 真实引擎处破损重构态——不应作为首条回流通道。
- **判定：真闭合度 60%**——主链闭合，回流审核 gate 缺失。

### K5 最小小节 ≠ TOC 三级（§6 步骤 3 + §8 假设 2）— 机制闭合 / 缺 TOC-less fallback
- **v0.2 做法**：§6 步骤 3「粒度经 `min_section_level` 可配置（按 `book_type` 选定：针灸=穴、本草=药、临床=证/方），默认候选 TOC 标题块，严禁重切生成新 Line」；§8 假设 2 裁决「可配置 + 按 book_type + 经 contentNodeId 挂载」。与建议 D 对齐。
- **闭合度：机制对，但漏了「TOC 缺失本」**。
  - 针灸 = 穴、本草 = 药、临床 = 证/方，方向正确。但**影印本 / 手写本常无结构化 TOC**（扫描古籍、稿本），既没有三级标题也没有可解析的 `contentNodeId` 来源。此时 `min_section_level` 可配置也救不了——没有标题可挂。
  - 必须补 fallback：①穴位名正则（`^\s*[一-龥]{1,4}穴\s*$` 或「经名 + 序号」）、②药名条目正则（`^\s*[一-龥]{2,4}\s*$` 配伍性味/归经标记）、③均无则整本挂单一 `ContentNode(source='raw_scan')`，并在 `BookRegistry` 标 `toc_missing=true` 提示下游。
- **判定：真闭合度 70%**——配置对，TOC-less fallback 缺。

### K6 方剂抽取责任 / 边界（§6 步骤 4）— 机制闭合
- **v0.2 做法**：`FormulaComposition` 置于 Archiver，且 §4 GlyphVerifier 明确只做字级校验（无语义解析），抽取职责隐式落在 Archiver/后处理。剂量保留原串（`各15`/`等分`/`适量`）已写。
- **闭合度：真闭合（隐式）**。建议显式加一句「方剂结构化抽取属 LLM/后处理职责，GlyphVerifier 不参与语义解析」，避免实现时越界。附方/变方/一方治多病的边界本次未展开，但属实现细节。
- **判定：真闭合度 90%**。

### K7 毒性药材剂量告警（§6 步骤 4）— 机制闭合 / 与原串保留矛盾
- **v0.2 做法**：§6 步骤 4「内置 `toxic_herbs.json` 打 `isToxic`，触发用量红线告警（细辛≤3g、附子须炮制）」。对应建议 E。
- **闭合度：机制对，但「原串保留」与「红线可判定」自相矛盾（新引入 N3）**。
  - 红线是**数值 / 布尔规则**（细辛 ≤3g、附子 must_processed=true），但方案同时要求「剂量保留原串（`各15`/`等分`/`适量`）」。若只留原串、不解析出数值，红线告警**无从触发**——`各15克`/`10–15克` 需解析后才能比 `≤3g`。
  - 必须二选一或双存：①原串 + 解析值（`dosageValue` 数值 + `unit` + `raw`）双列；②红线规则结构化（`max_grams`/`must_processed`）而非自然语言。建议 `FormulaIngredient` 同时存 `raw` 与解析后的 `numericValue/unit`，红线比对解析值。
- **判定：真闭合度 70%**——表与红线意图对，缺「原串 + 解析值双存」才能真触发告警。

### K8 中医专用形似混淆集（§4.1 + §4.2 步骤 4）— 机制闭合 / 覆盖不足
- **v0.2 做法**：§4.1「预计算形似混淆集（`confusion_set.json`：莪术↔我术、黄芩↔黄芪、半夏↔半下…）」；§4.2 步骤 4「命中形似混淆集 → FAIL，`auditSource=confusion`」。方向对（中医专用而非通用集）。
- **闭合度：机制对，但样例仅 3 对，覆盖严重不足（新引入 N4）**。
  - 我在 round3 K8 列出高频错对至少还应补：川芎↔川穹、栀子↔框子、紫菀↔紫苑、薤↔韭、葶苈↔亭历、**白朮↔白木**、茯苓↔茯芩、**巳↔已**（临床「已服」vs「巳服」影响服药时序）、**未↔末**（「未服」vs「末服」）。其中 **巳/已、未/末 属否定 / 时序语义，误判有临床风险**，优先级应高于药名类。
  - 建议 `confusion_set.json` 设 `severity`（negation > herb）字段，使 HumanGate 的 `severity=critical` 能优先排到否定 / 毒性相关混淆。
- **判定：真闭合度 70%**——框架对，需扩到 ≥15 对且带 severity。

## 三、v0.2 新引入的领域问题（N1–N4）

### N1 RARE 误放行风险（§4.2 步骤 3）
- RARE 免人工校对直接归档。若「中医候选字表」边界过宽（如「任意 CJK Ext-A/B 合法字即放行」），OCR 把「䗪虫」误识成另一合法但义错的 Ext-B 字时会被静默放过，违背原则 4「绝不静默放行错字」。
- **要求**：①RARE 仍须过字级置信门（如 `char_conf ≥ 0.7` 才免人工，低置信 RARE 转 UNCERTAIN 送检）；②RARE 行按 book 抽样（如每本抽 5%）进 QA 复核；③候选字表**仅含中医域名域字**（药名 / 穴名 / 正体字），不含通用生僻字。

### N2 闭环回填毒化（§6 步骤 5 + §9）
- 人工校正 → `CandidateSubmissionBatch` → 写回 `rare_allowlist/variant_map/confusion_set`。若无审核 gate，单点 OCR 错字经一次人工误校即固化进白名单，后续同错被 RARE/PASS 放行，错误被放大（正反馈毒化）。
- **要求**：回填设两级 gate——①人工校对员确认；②`rare_allowlist` 新增项进入「待审池」，需第二人复核或累计 N 次独立确认才升为正式放行。term_kb 路径因 kimi 破损态暂不启用。

### N3 剂量原串保留 vs 红线可判定矛盾（见 K7）
- 原串 + 解析值双存，红线比对解析值。否则细辛≤3g / 附子须炮制只是文档口号。

### N4 混淆集覆盖不足 + 缺否定语义优先（见 K8）
- 扩到 ≥15 对中医高频错对，带 `severity`，否定 / 时序类（巳/已、未/末）置 critical。

## 四、改进建议（具体、可落地）

**P1（最高优先，阶段 0 文档补）— 落地 4 个领域资源文件含种子数据**
- `variant_map.json`：首批内置 ≥200 条明确等价异体（朮→术、栝楼→括楼、藊→扁…）+ 候选异体档（提示不自动合并）。
- `confusion_set.json`：≥15 对中医高频错对，含 `severity`（negation/herb），必含 莪术↔我术、黄芩↔黄芪、半夏↔半下、白朮↔白木、巳↔已、未↔末、川芎↔川穹、栀子↔框子、紫菀↔紫苑、薤↔韭、葶苈↔亭历、茯苓↔茯芩。
- `toxic_herbs.json`：附子/乌头/马钱子/砒霜/蟾酥/斑蝥/生半夏/细辛…，每条带 `isToxic`、`max_grams`（细辛 3）、`must_processed`（附子 true）。
- 「中医候选字表（rare_allowlist）」：首批种子含 萆薢、䗪虫、蘡薁、虻虫、蛴螬、蜣螂、蝼蛄、鳢肠、蟾蜍、茺蔚子、楮实、蒺藜、菝葜 等，并随包内置而非冷启动为空。
- 这四条不落地，K1/K2/K7/K8 的「机制闭合」在首本实书上全部归零。

**P2（阶段 3）— RARE 加置信门 + 抽检（治 N1）**
- RARE 触发条件加 `char_conf ≥ 阈值`；低于阈值转 UNCERTAIN 送检。每本 RARE 行抽 5% 进 QA。候选字表限定中医域名域。

**P3（阶段 5）— 闭环回填加两级审核 gate（治 N2）**
- `CandidateSubmissionBatch` → 待审池 →（人工复核 / 累计 N 次确认）→ 正式 `rare_allowlist`；term_kb/khub 路径置「后续里程碑」，不阻塞主链。

**P4（阶段 5）— 剂量双存（治 N3）**
- `FormulaIngredient` 同时存 `dosageRaw`（各15/等分/适量）+ `dosageValue(numeric)` + `unit`；红线规则结构化（`max_grams`/`must_processed`）比对解析值。

**P5（阶段 5）— TOC-less fallback（治 K5 延伸）**
- 影印 / 手写本无 TOC 时：先试穴位 / 药名正则下钻到「穴 / 药」级；均无则整本挂单一 `ContentNode(source='raw_scan')`，`BookRegistry.toc_missing=true`。

**P6（阶段 3）— `variant_map` 两档制（治 K2 延伸）**
- 等价档自动合并，候选异体档仅提示不合并、送检，避免 菟丝子 误并为 兔子。
