# 古籍跨引擎分歧实测报告（25 本扩面 · 含 v4 第四轮）

> 文档版本：v2026-07-20
> 最后更新：2026-07-20
> 关联：`scripts/e2e_expand_books.py` · `kzocr/engine/cross_align.py` · 高分歧页视觉仲裁 §5.5
> 数据来源：`e2e_expand/summary.json`（16 本）+ `e2e_expand/summary_v4.json`（v4 追加 9 本），生成产物不入库

## 1. 背景与目的

KZOCR v0.7 的跨引擎分歧检测（`cross_align.run_cross_align`，PaddleOCR vs OvisOCR2-Q4_K_M token 级对齐）产出每页的 `Divergence`，按优先级分为**高**（high，含数字 / 中文数字 / 形近字分歧）与**普通**两类。这些分歧是后续人工校对（review 队列）与「高分歧页视觉仲裁」（§5.5 Box-Guided VL）的输入。

早前在 5 本古籍（各 20 页）上的采样曾给出一个**误导性的结论**——「分歧/页稳定在 7.8–15.6，是一个固定带」。本报告将样本扩大到 **16 本 / 840 页**，修正该结论；随后 **v4 第四轮（2026-07-20）再追加 9 本 / 337 页**，合并 **25 本 / 1177 页** 全量复盘（见 §7）。

## 2. 方法

- 脚本 `scripts/e2e_expand_books.py` 升级支持：
  - **增量合并 `--merge`**：只算未覆盖页，跳过已跑的 0–19 页，避免重复 OCR；
  - **每书落盘检查点**：长作业中途崩溃不丢进度；
  - **含空格文件名容错**（如 `名老中医之路 (全三册).pdf` 此前致 `ValueError`）。
- 渲染复用 orchestrator 全路径管线：`_pdf_page_to_numpy` + 版心裁切 `_crop_to_body` + 最长边 ≤ 2048 缩放，使双引擎分歧数字与 orchestrator 全路径严格可比（见提交 `c82453d`）。
- 5 本原书加深到 80 页（增量跳过 0–19），验证「20 页采样是否代表全书」；另 11 本新书各取 40 页。
- 数据来源 `e2e_expand/summary.json`（生成产物，不入库）。

## 3. 核心结论

- **跨引擎分歧/页并非固定带**，而是强烈依赖书籍版式、文字密度与扫描质量。16 本实测区间 **3.8–47.5/页**，跨度超 12×。
- 干净专著最低（疼痛妙方 3.8、学姚派 4.8）；密集验方类最高（验方新编下册 47.5、上册 17.1）。
- 16 本合计 **840 页 / 11701 分歧 / 2994 高分歧**；平均 **div/pg = 13.9**、**high/pg = 3.6**。
- **20 页采样对单书分歧/页具代表性**：5 本加深到 80 页后同量级波动（见 §5）。
- **逐页分布无单页灾难尖峰**：最高页（验方新编下册）每页 20–97、中位 46，属真实引擎分歧，非 OCR 整页失败。
- 高分歧页（high）占比随书变化明显（hi/pg 0.4–11.9），是 §5.5 视觉仲裁的主要分流对象。

## 6. 对流水线的含义

- **视觉仲裁负载不均衡**：high/pg 从 0.4（疼痛妙方）到 11.9（验方新编下册）差异巨大，§5.5 的 GLM-4V-Flash 调用量应按书动态评估，密集验方类需重点投入。
- **阈值不应按固定带设定**：分歧门控与采样率（`KZOCR_CONSENSUS_SAMPLE_RATE`）若假设「7.8–15.6 固定带」会误判低分歧书（如疼痛妙方）为异常、漏判高分歧书。
- **属于真实引擎分歧**：逐页分布平滑无尖峰，说明分歧来自双引擎识别差异本身，而非 OCR 整页失败，处理策略应为「逐字仲裁 + 人工复核」而非「整页重跑」。

### 7.3 关键发现

1. **分歧率仍由内容决定，无统一「带」**：v4 内 4.4（名老中医之路，传记散文干净）→ 20.3（264附子，单味药密集专论），跨度约 5×。新书以「神农本草经单味药专论 + 速查表」为主，整体偏密，与 §3 结论一致。
2. **high 占比分化明显，是更敏感的二级信号**：mi-678（45.2%）、全量中药速查总表（43.4%）近一半分歧为 high——双引擎在字符级分歧大且多难判字；附子（21.1%）、名老中医（20.9%）分歧虽多但多为易判差异（标点/异体字）。**high 占比比总分歧率更能预示「送 VL 仲裁后 unresolved 比例」**。
3. **前 5 页系统性偏低，采样易低估**：多数书 p0–p4 分歧很少（附子 p0–11 几乎全 0、乌头/半夏/虎掌前半本大量 0 分歧页），属封面/目录/凡例/方剂索引等非正文页。若 20 页采样从 p0 起且落于目录区，会系统性低估全书分歧率——**采样应从正文起始页起算，不能从 p0 起**。
4. **零分歧页 11.3% 多为非正文页**：38 页零分歧集中在各书前半本，印证上述结构。
5. **MuPDF xref 结构损坏告警**：全量中药速查总表 p30 附近报 `cannot find object in xref (64 0 R)`，脚本容错降级继续、39 页全部处理完。该 PDF 有轻微结构损坏，**建议后续扩面脚本对带此告警的源文件加渲染回检**，避免损坏页静默丢字。
6. **无灾难尖峰**：v4 最密页 61（附子 p14），仍属真实引擎分歧，非 OCR 整页失败。

### 7.4 对流水线含义的修订（补充 §6）

> 以下三项建议均已落地（2026-07-20，见 `9b87c7e` / `f5ce35d`）。

- **high 占比作为自动仲裁可信度的二级判据**：high 占比高的书（mi-678、速查表）送 §5.5 VL 仲裁时 unresolved 比例更高、需人工兜底更多；high 占比低的书分歧多为易判差异，可更激进自动接受。**→ 已落地**：`orchestrator._is_conservative(tally)` 在全书 high/总分歧 ≥ 0.40（样本 ≥10 页）时进入保守模式，`_arbitrate_high_divergences(conservative=True)` 即便 VL 明确接受也全部留人工复核。
- **采样协议修正**：扩面/抽检脚本应跳过封面目录区，从正文首页起算 N 页，否则分歧率数字偏低失真。**→ 已落地**：`scripts/e2e_expand_books.py` 新增 `--body-start N`，采样从正文起始页起算。
- **PDF 健康度回检**：对带 MuPDF xref 告警的源文件，渲染后比对文本非空，避免损坏页静默丢字。**→ 已落地**：`render_page` 返回 `(img, healthy)`，fitz 文本层缺失且图像非空白时标记 `healthy=False`，`count_book` 收集 `render_warnings` 供核查。
- **耗时稳定**：9 本单页均值 ~21s，与 PaddleOCR 基准（21s/页）一致，证明扩面流程在长作业下无退化。

## 8. 复现

```bash
# 增量合并模式（跳过已跑 0–19 页），每书输出检查点
python scripts/e2e_expand_books.py --merge -o e2e_expand/summary.json

# v4 第四轮：追加书目清单（9 本），输出独立 summary
python scripts/e2e_expand_books.py --list e2e_expand/books_expand_v4.txt -o e2e_expand/summary_v4.json
```

生成产物 `e2e_expand/summary.json`（16 本）与 `e2e_expand/summary_v4.json`（v4 追加 9 本）各含每书逐页 `div` / `high` 明细，不入库；合并全量 = 25 本 / 1177 页 / 15688 分歧。
